# Issue 25640 investigation

- Issue: https://github.com/strapi/strapi/issues/25640

## Finding

- Reported relation-cleanup bug not reproduced.
- `document-manager.deleteMany` already delegated to `strapi.documents(uid).delete(...)`.
- That single-document path already cleans relations/components.
- The misleading part was the stale `// FIXME: handle relations` comment in:
  - `packages/core/content-manager/server/src/services/document-manager.ts`
- Real bug found nearby:
  - bulk delete could overcount when duplicate `documentId`s reached the service.

## Fix

- removed stale `FIXME`
- kept bulk delete on document-service delete path
- deduped `documentIds`
- computed `populate` once
- counted only documents that actually deleted entries
