# Issue #26571 — session expiry unsaved-changes guard

https://github.com/strapi/strapi/issues/26571

## Reply to the reporter

The mid-edit logout while actively typing was addressed in #26680 (5.48+). This PR covers the remaining gap: when a session-expiry logout still happens (idle timeout, `maxSessionLifespan`, failed renewal), unsaved Content Manager edits are no longer discarded silently.

Before this change, clearing the JWT made `PrivateRoute` render `<Navigate to="/auth/login" />`, unmounting the edit form before its route `Blocker` could run. The auth layer now checks a shared unsaved-changes registry (registered by `Form`/`Blocker`) and shows the standard confirmation dialog before clearing auth state.

## Recommendation

Merge after review. Closes the actionable part of #26571; `maxSessionLifespan` hard-ceiling data loss and draft persistence remain separate product decisions.

## Supporting detail

- **Proof tests:** `packages/core/admin/admin/src/utils/tests/unsavedChangesRegistry.test.ts`
- **Root cause:** token clear → `PrivateRoute` unmount → form state lost; `baseQuery` previously dispatched `logoutAction()` before the React guard could run
- **Fix:** registry + `AuthProvider` dialog; `baseQuery` defers to `triggerSessionExpired()` only
- **Related:** #26680 (active-session renewal), #26165 (idle redirect)
